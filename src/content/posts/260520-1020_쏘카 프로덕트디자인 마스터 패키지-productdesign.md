# 쏘카 프로덕트 디자인, 1천만 다운로드 앱을 만든 실무 노하우 A to Z

## 🧐 기존 기능 개선, 언제나 어렵죠?
![쏘카 앱 화면 예시](https://cdn.day1company.io/prod/uploads/202402/083918-569/socar-productdesignall-detail-01.png)
분명 시키는 대로 했는데 왜 내 작업물은 어딘가 부족할까요? 사용자 경험을 개선하는 프로젝트는 의외로 간단한 관점에서 출발할 때가 많습니다.

---

## 💡 실무 퀄리티를 좌우하는 3가지 핵심 포인트

### 1. 페인 포인트 발굴: 사용자 문제점을 '진짜'로 정의하기
단순히 기능 추가나 UI 변경에 그치지 않고, 사용자가 겪는 핵심 불편함(Pain Point)을 정확히 찾아내는 것이 중요합니다. VOC 분석, 사용자 인터뷰 등 객관적인 데이터를 기반으로 문제의 근본 원인을 파악하는 방법론을 익힙니다.

### 2. UX Flow 설계: 복잡한 여정을 간결하게 만들기
사용자 경험을 설계할 때는 단순히 화면 전환뿐 아니라, 각 단계별 사용자 행동 패턴을 예측하고 최적의 흐름을 만드는 것이 핵심입니다. 쏘카의 성공 사례를 통해 복잡한 여정을 어떻게 간결하고 직관적으로 설계하는지 배울 수 있습니다.

### 3. 디자인 시스템 구축: 일관성과 효율성을 동시에 잡는 법
새로운 기능을 만들거나 기존 기능을 개선할 때, 통일된 디자인 원칙과 재사용 가능한 컴포넌트 구축은 필수입니다. 쏘카의 경험을 통해 효율적인 디자인 시스템을 구축하고, 브랜드 일관성을 유지하며 개발 리소스까지 절감하는 노하우를 습득합니다.

실제 쏘카 앱을 기반으로, UX 설계부터 디자인 리뷰까지 전 과정을 밀도 있게 다룹니다. 기존 디자인 원칙과 새로운 프로덕트의 특성을 모두 고려한 디자인 원칙 구축 방법, 팀원들과의 효과적인 디자인 리뷰를 통해 프로덕트 완성도를 높이는 노하우를 배웁니다. 특히, 사용자 관점에서 문제점을 발굴하고 비즈니스 목표 달성을 위한 디자인 전략 수립 과정을 배우는 것은 실무에서 바로 적용 가능한 핵심 스킬입니다.

## 💬 자주 묻는 질문 (FAQ)

**Q. [Onboarding] Socar 쏘카 프로덕트디자인 마스터패키지: 신규 기능 생성 시, 사용자 니즈를 어떻게 정확하게 파악하나요?**

VOC 분석, 사용자 설문, 실제 사용 데이터 분석 등 다양한 방법론을 통해 사용자의 잠재된 니즈와 불편함을 발굴하는 과정을 상세히 배웁니다. 단순히 표면적인 요구사항이 아닌, 근본적인 문제 해결에 집중하는 방법론을 익힙니다.

**Q. [Onboarding] Socar 쏘카 프로덕트디자인 마스터패키지: 프로덕트 디자인 성과를 어떻게 측정하고 개선 방향을 설정하나요?**

전환율, 계약률, 사용률 등 정량적인 지표를 활용하여 프로덕트의 성과를 객관적으로 측정하고, 측정된 데이터를 기반으로 구체적인 개선 방안을 도출하는 방법을 학습합니다. 성공적인 서비스는 결국 데이터에 기반한 개선이 중요합니다.

**Q. [Onboarding] Socar 쏘카 프로덕트디자인 마스터패키지: 외부 도메인 적용 시, UI 최적화는 어떻게 접근해야 하나요?**

외부 도메인의 기존 화면 문제점을 정량적/정성적 데이터로 분석하고, 사용자 니즈를 반영한 정보 흐름과 UI 개선을 통해 사용자 경험을 지키면서 UI를 최적화하는 방법을 배웁니다. 개발 효율화와 디자인 시스템 정의를 통해 사용 여정 동작 수를 간소화하는 과정까지 포함합니다.
