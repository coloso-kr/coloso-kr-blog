# 피그마 프로토타입, 어렵지 않아요! 한승헌 강사가 알려주는 핵심 기능 총정리

## 🧐 혹시 이런 고민 하고 계신가요?

"피그마 프로토타입, 기능은 많은데 어디서부터 시작해야 할지 모르겠어요.", "클릭, 드래그 같은 기본적인 인터랙션은 알겠는데, 더 다양하고 동적인 효과를 구현하고 싶어요.", "애니메이션 효과를 자연스럽게 적용하는 방법이 궁금해요."

UI/UX 디자인의 중요성이 날로 커지면서 프로토타이핑 툴인 피그마의 활용도 또한 크게 증가했습니다. 특히 프로토타입 기능이 강화되면서 더욱 생동감 넘치는 디자인을 구현할 수 있게 되었는데요. 하지만 처음 피그마 프로토타입 기능을 접하는 분들이라면 다양한 옵션과 기능들 사이에서 길을 잃기 쉽습니다.

본 블로그 포스팅에서는 콜로소의 한승헌 강사님의 강의 내용을 바탕으로, 피그마 프로토타입의 필수 기능부터 고급 활용법까지 상세하게 안내하여 여러분의 디자인 역량을 한층 더 끌어올릴 수 있도록 돕겠습니다.

## 💡 피그마 프로토타입 해결의 실마리

최근 UI/UX 디자인의 수요와 니즈가 증가하면서 피그마 툴 또한 프로토타이핑 기능이 대폭 강화되었습니다. 한승헌 강사님은 강의를 통해 피그마에서 제공하는 거의 모든 프로토타입 기능을 다루며, 실전 예제와 함께 학습할 수 있도록 구성했습니다.

- **상황 진단**: 복잡하게만 느껴지는 프로토타입 패널과 다양한 인터랙션 옵션 때문에 프로토타이핑 구현에 어려움을 겪는 상황을 진단합니다.
- **핵심 원리**: 피그마 프로토타입 패널의 기본 구조와 각 인터랙션(On click, On drag, While hovering 등)의 작동 방식을 명확히 이해하는 것이 중요합니다.
- **실무 가이드**: 기본적인 탭 인터랙션부터 스크롤, 노트 프로토타이핑, 컴포넌트 프로토타이핑까지, 다양한 예제를 통해 실제 적용 방법을 익힙니다.

## 🛠️ 현직자가 짚어주는 피그마 프로토타입 활용 디테일

피그마 프로토타입의 강력함을 제대로 활용하기 위해서는 각 기능을 정확히 이해하고 전략적으로 사용하는 것이 중요합니다. 한승헌 강사님은 단순히 기능을 나열하는 것을 넘어, 각 인터랙션이 어떤 상황에서 효과적인지, 그리고 어떻게 하면 더 자연스럽고 매력적인 애니메이션을 만들 수 있는지 상세히 설명합니다.

### 1. 프로토타입 패널 살펴보기

프로토타입 제작은 우측의 'Prototype' 탭을 통해 이루어집니다. 프레임을 선택하면 'Flow starting point', 'Interactions', 'Scroll behavior' 등의 옵션이 나타납니다.

- **Scroll behavior**: 스크롤 방향(상하, 좌우, 양방향)을 설정합니다.
- **Interactions**:
    - **On click**: 클릭 시 동작
    - **On drag**: 드래그 시 동작
    - **While hovering**: 마우스 오버 시 동작
    - **While pressing**: 마우스 클릭 유지 시 동작
    - **Key/Gamepad**: 키보드 또는 게임패드 입력 시 동작
    - **After delay**: 일정 시간 후 동작

이러한 인터랙션에 이어지는 액션으로는 'Navigate to', 'Back', 'Set variable', 'Conditional', 'Scroll to', 'Open link', 'Open overlay', 'Swap overlay', 'Close overlay' 등이 있습니다.

### 2. Smart Animate 활용하기

'Smart animate' 기능은 좌측과 우측 프레임 간의 차이점을 분석하여 자동으로 애니메이션을 생성해주는 강력한 기능입니다. 요소의 위치, 크기, 색상 등이 변경될 때 이 기능을 활용하면 복잡한 애니메이션 작업 없이도 자연스러운 전환 효과를 만들 수 있습니다. 예를 들어, 작은 사각형이 클릭 후 커다란 사각형으로 변하는 애니메이션을 쉽게 구현할 수 있습니다.

### 3. Overlay 기능으로 팝업 및 모달 구현

'Open overlay' 기능을 사용하면 현재 화면 위에 새로운 프레임을 팝업 또는 모달 형태로 띄울 수 있습니다. 'Move in', 'Dissolve' 등 다양한 애니메이션 효과와 함께 'Close when clicking outside' 옵션을 통해 사용자가 오버레이 바깥 영역을 클릭했을 때 자동으로 닫히도록 설정할 수 있어 사용자 경험을 향상시킬 수 있습니다.

### 4. 컴포넌트와 인터랙션의 결합

버튼과 같은 컴포넌트에서는 'While hovering' 또는 'While pressing' 인터랙션을 통해 상태 변화를 표현할 수 있습니다. 예를 들어, 버튼에 마우스를 올렸을 때 색상이 변하거나, 눌렀을 때 눌린 상태의 디자인으로 전환되는 등의 효과를 'Change to' 액션을 통해 구현할 수 있습니다. 이는 디자인 시스템 구축 시 일관된 사용자 경험을 제공하는 데 중요한 역할을 합니다.

### 5. Jitter를 활용한 포트폴리오 모션 제작

최종적으로는 'Jitter'와 같은 서비스를 활용하여 제작된 프로토타입에 동적인 모션을 추가하고, 이를 포트폴리오나 드리블 등에 게시하는 방법까지 다룹니다. Jitter는 단순하고 간편하게 클릭 몇 번으로 웹사이트에 적용할 수 있는 모션을 만들 수 있도록 돕는 툴입니다.

## 💬 FAQ: 자주 묻는 질문들

**Q1. 피그마 프로토타입에서 'Smart animate' 기능은 어떻게 작동하나요?**
A1. 'Smart animate'는 두 프레임 간의 레이어 이름과 속성 차이를 분석하여 자동으로 중간 애니메이션 프레임을 생성해주는 기능입니다. 요소의 위치, 크기, 색상 등이 자연스럽게 전환됩니다.

**Q2. 'Open overlay' 기능을 사용할 때 주의할 점이 있나요?**
A2. 오버레이는 일반적으로 화면 전체를 프레임으로 만들기보다는, 보여주고자 하는 오버레이의 크기에 맞춰 프레임을 생성하는 것이 효과적입니다. 또한, 'Close when clicking outside' 옵션을 통해 사용자 편의성을 높일 수 있습니다.

**Q3. 버튼의 호버링(Hovering) 효과를 어떻게 만들 수 있나요?**
A3. 버튼을 컴포넌트로 만들고, 인터랙션에서 'While hovering'을 선택한 후 'Change to' 액션을 사용하여 호버링 시 변경될 버튼의 디자인 상태로 연결하면 됩니다.

**Q4. 'After delay' 인터랙션은 어떤 경우에 유용하게 사용될 수 있나요?**
A4. 페이지 로딩 후 자동으로 특정 모션이 재생되거나, 특정 시간이 지난 후 다음 화면으로 이동하는 등, 사용자 조작 없이 자동으로 작동해야 하는 인터랙션에 활용할 수 있습니다.

**Q5. 'Scroll behavior' 옵션에는 어떤 것들이 있나요?**
A5. 'Scroll behavior'는 스크롤 방향을 설정하는 것으로, 'Vertical' (상하), 'Horizontal' (좌우), 'Both' (양방향) 옵션을 제공합니다.

**Q6. 프로토타입 제작 시 'Design System'을 활용하는 것이 왜 중요한가요?**
A6. 디자인 시스템을 활용하면 일관된 디자인 언어를 유지하고, 컴포넌트 재사용을 통해 작업 효율성을 높일 수 있습니다. 프로토타입 제작 시에도 이러한 컴포넌트 기반 인터랙션이 용이해집니다.
